The proxy is thin (transport + minimal origin context). The Agent (Bedrock) orchestrates the foundation model and action groups. Deterministic facts (IATA/time/pricing) come from tools/Lambda — per AWS Bedrock guidance.

Agent, Versions, Aliases & Models

The foundation model used by the Agent is configured in AWS Bedrock per version. Aliases (e.g., Paul/Bianca/Gina) point to specific versions. This repo does not hard‑code a model; fetch it from Bedrock to avoid drift.

Why versions: versions snapshot instructions, tools and model selection; aliases enable safe promotion/rollback — a common Bedrock pattern.
